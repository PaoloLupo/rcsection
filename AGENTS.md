# AGENTS.md - RCSection

## Project Overview

RCSection is a Typst package for drawing reinforced concrete structural sections (beams, columns, slabs). It consists of:

1. **Typst Frontend** (`src/`) – Imports a WASM plugin, exposes user-facing functions.
2. **WASM Plugin** (`plugin/`) – Written in Rust, parses RCS domain-specific language and generates geometric primitives.
3. **Renderer** (`src/draw.typ`) – Uses CETZ (Typst drawing library) to render primitives into PDF.

## Architecture

```
User writes RCS code in Typst raw block
           |
           v
   Typst frontend (rcsection.typ)
           |
           v
   WASM plugin (parser.wasm)
     - CBOR in / CBOR out
     - LALRPOP grammar parses RCS syntax
     - Rust geometry engine generates primitives
           |
           v
   CETZ renderer (draw.typ)
     - Rect, Circle, Path, Text, Dimension, LeaderLine
           |
           v
   PDF output
```

## Build & Development

### Prerequisites

- Rust (with `wasm32-unknown-unknown` target)
- `just` (task runner)
- `typst` CLI
- `tt` (tytanic) for visual regression tests
- Python 3 (for packaging scripts)

### Common Commands

```bash
# Build WASM plugin and copy to src/
just plugin

# Run Rust unit tests + Typst visual tests
just test

# Update test reference images
just update

# Compile documentation
just doc

# Package for local or preview install
just install
just install-preview
```

## Coding Conventions

### Rust (plugin/)

- Edition 2024.
- Run `cargo clippy --all-targets --all-features` before committing.
- Prefer `matches!` over simple `match` returning bool.
- Extract helpers for duplicated logic (e.g., `get_section_dims`, `get_cover`).
- Avoid `.unwrap()` on user input; propagate errors or use sensible defaults.
- Use `#[cfg(test)] mod tests` in the same file for unit tests.

### Typst (src/)

- Use kebab-case for function names (`draw-primitive`, `parse-stroke`).
- Keep functions focused; extract helpers for repeated stroke/fill logic.
- Use named arguments with sensible defaults.

## Testing Strategy

1. **Rust unit tests** – `cargo test` covers parser and geometry engine.
2. **Typst visual tests** – `tt run` compares rendered PNGs against references.
3. **CI** – GitHub Actions runs `cargo clippy`, WASM build, and full test suite.

## Important Notes for Agents

- **Never commit `parser.wasm` without rebuilding it** via `just plugin`.
- **LALRPOP generates `grammar.rs`** in `target/.../out/`. Do not edit it directly.
- **The `BTreeSet<String>` hack for stirrup positions was removed** in favor of `Vec<f64>`.
- **Color mapping** (`get_color_for_size`) uses a single normalized match; avoid duplicating logic.
